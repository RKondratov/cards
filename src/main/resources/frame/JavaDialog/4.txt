Базовые функциональные интерфейсы.
|
1. Predicate - функциональный интерфейс для проверки некоторого условия.
Метод boolean test(Tt).
2. Consumer - функциональный интерфейс, который принимает в качестве входного аргумента
объект типа Т, совершает некоторые действия, но при этом ничего не возвращает.
Метод void accept(Tt).
3. Supplier - функциональный интерфейс, который не принимает никаких аргументов,
но возвращает некоторый объект типа Т.
Метод Т get().
4. Function - функциональный интерфейс, который принимает аргумент Т, приводит его к
объекту типа R и возвращает его как результат.
Метод R apply(Tt).
5. UnaryOperator - функциональный интерфейс, принимает в качестве параметра объект
типа Т, выполняет над ним некоторые операции и возвращает результат операции в виде того же типа Т.
Метод T apply(Tt).
